## Problem
The build fails at the `pnpm install --frozen-lockfile --prefer-offline` step with `ERR_PNPM_LOCKFILE_CONFIG_MISMATCH`. The `overrides` configuration in `pnpm-workspace.yaml` does not match what is recorded in `pnpm-lock.yaml`. The Replit Agent committed a regenerated lockfile that is out of sync with the overrides. Railpack auto-detects pnpm from the workspace setup and runs `pnpm install --frozen-lockfile`, which rejects the mismatched lockfile.

## Diagnosis
`pnpm-workspace.yaml` defines extensive `overrides` that exclude platform-specific optional dependencies (esbuild, lightningcss, @tailwindcss/oxide, rollup, @expo/ngrok-bin binaries). The lockfile was regenerated by the Replit Agent in commit fb44fa17 but does not reflect these overrides. When Railpack runs `pnpm install --frozen-lockfile`, pnpm detects the mismatch and exits with code 1, producing the error: `ERR_PNPM_LOCKFILE_CONFIG_MISMATCH: Cannot proceed with the frozen installation. The current "overrides" configuration doesn't match the value found in the lockfile`.

## Fix
This fix requires running the pnpm CLI and cannot be implemented as a pure file edit:
1. In the project root, run `pnpm install` (without `--frozen-lockfile`) to regenerate `pnpm-lock.yaml` against the current overrides in `pnpm-workspace.yaml`.
2. Verify the updated lockfile reflects the overrides (platform-specific binary exclusions for esbuild, lightningcss, @tailwindcss/oxide, rollup, @expo/ngrok-bin).
3. Commit and push the updated `pnpm-lock.yaml` to the `main` branch.
